Overview

Replace the current modal-based request details interface with a new tabbed interface that provides better organization and workflow management.

Current Implementation Issues:
  • Modal dialog limits screen real estate
  • No organized view of tasks, results, and milestones
  • Action buttons scattered in different locations
New Implementation Goals:
  • Tabbed interface for better organization
  • Centralized action buttons
  • Comprehensive milestone tracking

Action Buttons Specifications

1. Assign Button

  • Visibility: Processor, Manager, Admin
  • Enabled when: Any status, any user
  • Action: Opens assignment modal
Assign Request Modal

2. Start Button

  • Visibility: Requestor, Processor, Manager, Administrator
  • Enabled when: Status is "Pending" AND (assigned user is null OR current user)
  • Action: Shows confirmation modal
  • Result: Sets status to "In Progress", assigns to current user if unassigned
Start Confirmation

Are you sure you want to start this request?

3. Cancel Button

  • Visibility: Processor, Manager, Admin
  • Enabled when: Status is "Pending" OR "In Progress"
  • Action: Shows confirmation modal
  • Result: Sets status to "Cancelled"
Cancel Request

Are you sure you want to cancel this request?

4. Complete Button

  • Visibility: Assigned user, Processor, Manager, Admin
  • Enabled when: Status is "In Progress"
  • Action: Requires results to be saved first
  • Result: Sets status to "Completed"

Available from Results tab after saving results

Milestone Recording Rules

Automatic Milestones

  • Assigned: When assigned to a user
  • Started: When status changes to "In Progress"
  • Completed: When status changes to "Completed"

Button Enabling Logic

Button Enabled Conditions Additional Logic
Start Status = "Pending" AND (assigned_user IS NULL OR assigned_user = current_user)
Assign Always enabled For authorized roles only
Cancel Status = "Pending" OR "In Progress" For authorized roles only
Complete Status = "In Progress" AND results have been saved

Implementation Notes

Frontend Requirements
  • Bootstrap 5.x for responsive design
  • Tab navigation with active state management
  • Modal dialogs for confirmations
  • Real-time status updates
  • Form validation for required fields
Backend Requirements
  • REST API endpoints for all actions
  • Role-based authorization middleware
  • Automatic milestone creation triggers
  • Status validation before state changes
  • Audit logging for all actions
API Endpoints Needed
GET    /api/requests/{id}           - Get request details
PUT    /api/requests/{id}/assign    - Assign request
PUT    /api/requests/{id}/start     - Start request
PUT    /api/requests/{id}/complete  - Complete request  
PUT    /api/requests/{id}/cancel    - Cancel request
GET    /api/requests/{id}/tasks     - Get request tasks
POST   /api/requests/{id}/tasks     - Add new task
GET    /api/requests/{id}/milestones - Get milestones
PUT    /api/requests/{id}/results   - Save results
