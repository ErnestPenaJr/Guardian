# Guardian MVP — SQL Server → PostgreSQL migration

Scripts and generated SQL for moving the Guardian database off SQL Server
(Azure SQL) onto PostgreSQL.

## Files

| File | Committed? | Purpose |
|------|-----------|---------|
| `../scripts/mssql-to-postgres.cjs` | ✅ yes | Generator. Connects to a SQL Server source, introspects the `GUARDIAN` schema, and regenerates `01_schema.sql` + `02_seed.sql`. |
| `01_schema.sql` | ✅ yes | PostgreSQL DDL: schema, 55 tables, PKs, uniques, 85 indexes, check constraints, and 66 **DEFERRABLE** foreign keys. |
| `02_seed.sql` | ❌ **git-ignored** | Full data clone (every table) as batched `INSERT`s. **Contains real PII + password hashes — treat as a secret.** |

## Regenerating

The generator reads `DATABASE_URL` (a Prisma `sqlserver://…` string). With no
env var it falls back to the **staging** connection (`GUARDIAN-DEV`).

```bash
# Full regen (schema + data) from staging into ./postgres
node scripts/mssql-to-postgres.cjs --out ./postgres

# Schema only (no data dump)
node scripts/mssql-to-postgres.cjs --schema-only --out ./postgres

# Point at a specific source
DATABASE_URL="sqlserver://host:1433;database=DB;user=U;password=P;encrypt=true" \
  node scripts/mssql-to-postgres.cjs
```

Flags: `--schema <name>` (default `GUARDIAN`), `--out <dir>` (default `./postgres`),
`--schema-only`, `--batch <n>` (rows per INSERT, default 200).

## Loading into PostgreSQL

```bash
# 1. create the target database
createdb guardian            # or: psql -c "CREATE DATABASE guardian;"

# 2. schema first, then data
psql -d guardian -v ON_ERROR_STOP=1 -f postgres/01_schema.sql
psql -d guardian -v ON_ERROR_STOP=1 -f postgres/02_seed.sql
```

`02_seed.sql` runs inside a single transaction with `SET CONSTRAINTS ALL
DEFERRED`, so foreign keys are checked once at `COMMIT` — no load-order juggling
required. The tail of the file resyncs every identity sequence to its current
max so new inserts continue from the right value.

This was verified end-to-end against `postgres:16`: all 55 tables load, row
counts match the source exactly, `bytea` attachment blobs survive intact,
`bit`→`boolean` columns type correctly, and identity sequences resume past the
imported max.

## SQL Server → PostgreSQL conversions handled

- **Identifiers** are quoted and case-preserved (`"GUARDIAN"."USERS"`) — a 1:1 copy.
- **Types**: `bit`→`boolean`, `int`→`integer`, `nvarchar/varchar(MAX)`→`text`,
  `varbinary(MAX)`→`bytea` (emitted as `decode('…','hex')`), `datetime/datetime2`→`timestamp`,
  `uniqueidentifier`→`uuid`, `numeric(p,s)` preserved, `money`→`numeric(19,4)`.
- **Identity**: `IDENTITY(seed,inc)` → `GENERATED BY DEFAULT AS IDENTITY (START WITH … INCREMENT BY …)`
  (allows explicit-value inserts during seed; resynced afterward).
- **Defaults**: `getdate()/sysdatetime()`→`now()`, `getutcdate()`→`timezone('utc', now())`,
  `newid()`→`gen_random_uuid()` (needs the `pgcrypto` extension, created by the schema file),
  bit `0/1`→`false/true`, and temporal `CONVERT([datetime2],'…')` literals (7→6 fractional digits).
- **Check constraints**: `[col]`→`"col"`, `LEN()`→`length()`, `DATALENGTH()`→`octet_length()`,
  `ISNULL()`→`coalesce()`, and bit comparisons `"FLAG"=(1)`→`"FLAG" = true`.

## Caveats / not carried over

- **System-versioned temporal tables.** PostgreSQL has no native system
  versioning. The `*_HISTORY` tables are created as plain tables and their data
  is preserved, but the `SysStartTime`/`SysEndTime` period columns on the *live*
  tables are emitted as ordinary timestamp columns (their data is regenerated by
  defaults, not copied). If you need ongoing history capture, add triggers or a
  temporal extension after loading.
- **App-side queries.** The application's raw SQL (`server.cjs` etc.) references
  unquoted names like `GUARDIAN.USERS`, which PostgreSQL folds to lowercase and
  will **not** match the quoted, upper-case tables created here. Switching the
  app to PostgreSQL is a separate task (Prisma `provider`, query syntax, etc.).
- **`pgcrypto`** extension is required for `gen_random_uuid()` defaults; the
  schema file creates it (`CREATE EXTENSION IF NOT EXISTS pgcrypto`).
